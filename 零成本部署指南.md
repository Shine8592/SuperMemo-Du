# 零成本部署指南

> 使用免费资源和开源工具，零成本部署嘟嘟超级记忆系统

## 📋 前提条件

### 硬件要求
- 最小：1核 CPU, 1GB RAM, 10GB 存储
- 推荐：2核 CPU, 2GB RAM, 20GB 存储
- 支持平台：Linux x86_64/arm64 (Ubuntu/Debian/CentOS等)

### 软件要求
- Git
- Python 3.8+
- 基础 Linux 命令行工具
- 互联网连接

## 🚀 一键部署脚本

### 方案1：完全自动化（推荐）
```bash
# 一条命令完成部署
curl -fsSL https://raw.githubusercontent.com/Shine8592/SuperMemo-Du/main/deploy.sh | bash
```

### 方案2：手动逐步部署
```bash
# 1. 克隆仓库

# 4. 启动服务

```

## 🔧 手动部署详细步骤

### 第一步：获取源代码
```bash
git clone https://github.com/Shine8592/SuperMemo-Du.git
cd SuperMemo-Du
```

### 第二步：安装依赖
```bash
# Python依赖
pip3 install -r requirements.txt

# 系统依赖（Ubuntu/Debian示例）
sudo apt-get update
sudo apt-get install -y \
    curl \
    wget \
    unzip \
    jq \
    sqlite3
```

### 第三步：配置环境
```bash
# 复制示例配置
cp config.example.yaml config.yaml

# 编辑配置文件
nano config.yaml

# 关键配置项说明：
# storage:
#   hot_size_mb: 512      # 热存储大小 (MB)
#   warm_size_mb: 2048    # 温存储大小 (MB) 
#   cold_path: ./cold     # 冷存储路径
# 
# memory:
#   short_term_hours: 24  # 短期记忆保留时间
#   long_term_threshold: 0.7  # 长期记忆阈值
# 
# api:
#   port: 8080            # API服务端口
#   host: 0.0.0.0         # 绑定地址
```

### 第四步：初始化系统
```bash
# 创建必要目录
mkdir -p ./hot ./warm ./cold ./backups ./logs

# 设置权限
chmod +x *.sh
chmod 600 config.yaml  # 保护敏感配置

# 运行初始化
./init.sh
```

### 第五步：启动服务
```bash
# 前台运行（调试用）
python3 -m suprmemo.main

# 后台运行（生产环境）
./start.sh

# 使用 systemd（Linux服务）
sudo cp suprmemo.service /etc/systemd/system/
sudo systemctl daemon-reload
sudo systemctl start suprmemo
sudo systemctl enable suprmemo
```

## 💰 零成本资源利用方案

### 存储方案
| 存储层 | 零成本选择 | 说明 |
|--------|------------|------|
| **热存储** | /dev/shm (tmpfs) | 内存文件系统，最快但重启后丢失 |
| **温存储** | 本地磁盘 HDD | 容量大，成本低，速度一般 |
| **冷存储** | GitHub 私有仓库 + 归档分支 | 免费私有仓库，版本历史天然备份 |

### 网络和计算
- **计算资源**：家庭旧电脑、树莓派、VPS免费层
- **网络带宽**：家庭宽带足够满足个人使用需求
- **电力成本**：现有设备增量开销几乎为零

### 具体实施示例
#### 方案A：树莓派 Zero 2 W + 32GB SD卡
- 成本：约 $15 一次性
- 功能：完整运行嘟嘟超级记忆系统
- 适用场景：个人助理、知识管理

#### 方案B：老笔记本 + 自由空间
- 成本：基本为零（利用闲置设备）
- 功能：性能更强，可扩展性好
- 适用场景：家庭服务器、小团队使用

#### 方案C：纯云免费方案
- GitHub Codespaces：免费额度可用于开发测试
- Railway.app/Render.com：有免费层的应用托管
- 注意：纯方案可能有使用限制和冷启动延迟

## 🔐 安全加固（零成本）

### 基础安全
```bash
# 1. 更改默认端口（如果暴露公网）
# 2. 创建非root用户运行服务
sudo useradd -r -s /bin/false suprmemo
sudo chown -R suprmemo:suprmemo /opt/suprmemo

# 3. 基本防火墙规则（ufw）
sudo ufw allow from 192.168.1.0/24 to any port 8080
sudo ufw enable

# 4. SSH密钥登录而非密码
```

## 📊 性能调校建议

### 内存优化
```yaml
# 在 config.yaml 中调整
memory:
  short_term_limit_mb: 256    # 根据实际内存调整
  long_term_batch_size: 100   # 批处理大小
  gc_interval_hours: 6        # 垃圾回收间隔
```

### 存储优化
```yaml
storage:
  compression_enabled: true   # 启用压缩
  compression_level: 6        # 平衡压缩率和速度
  deduplication_enabled: true # 启用去重
```

## 🛠️ 常见问题解答

### Q: 系统需要多少磁盘空间？
A: 取决于使用强度。建议预留：
- 初始部署：2GB
- 每月增量：约500MB-2GB（取决于对话量）
- 1年累计：约10GB-20GB

### Q: 如何备份和恢复？
A: 系统自带一键备份和恢复功能：
```bash
# 备份（推送到配置的远程仓库）
./backup.sh

# 恢复（从最近备份）
./restore.sh

# 查看所有备份点
./backup.sh --list
```

### Q: 系统会变慢吗？
A: 嘟嘟超级记忆系统设计有自动清理和优化机制：
- 短期记忆自动滚窗
- 长期记忆定期压缩和去重
- 自动垃圾回收（默认每6小时）
- 索引定期重建保持性能

### Q: 如何升级到新版本？
A: 由于采用零成本方案，建议：
```bash
# 1. 备份当前状态
./backup.sh

# 2. 拉取最新代码
git pull origin main

# 3. 检查更新日志
git log HEAD~1..HEAD --oneline

# 4. 运行升级脚本（如果有）
chmod +x upgrade.sh
./upgrade.sh

# 5. 重启服务
./restart.sh
```
